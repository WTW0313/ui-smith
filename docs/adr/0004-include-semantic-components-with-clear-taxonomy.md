# Include Semantic Components With Clear Taxonomy

UI Smith v1 will include Semantic Components alongside Primitive-Backed Components so the First Component Set can cover practical app UI needs such as text areas and alerts. The generated docs and tests must distinguish which components rely on Base UI Primitive behavior and which rely on native HTML and ARIA semantics.

## Consequences

UI Smith can generate a more useful initial Component Kit, but it must avoid implying that every component has Base UI-provided behavior. Accessibility claims should be scoped to the component category and backed by Verification Evidence.
